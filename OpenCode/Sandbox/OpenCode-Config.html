Configuration Guide

OpenCode is configured primarily through opencode.jsonc (main settings) and tui.json (UI and keybindings). This guide provides a detailed overview of all available options.

1. Core AI & Model Configuration

Define which AI providers and models OpenCode uses to power its intelligence.

Option Type Description
model string Active model ID in provider/model format (e.g., anthropic/claude-3-5-sonnet).
small_model string Faster model for background tasks like session title generation.
enabled_providers string[] If set, only these providers will be loaded.
disabled_providers string[] Providers to ignore during automatic loading.
provider record Custom overrides: apiKey, baseURL, timeout, and models.

Model Object Detailed Configuration

When overriding models within the provider configuration, you can use the Model object to define precise capabilities, limits, and costs.

Option Type Description
id string The unique identifier for the model.
name string The human-readable display name.
family string The model family (e.g., Claude 3.5).
release_date string The official release date of the model.
attachment boolean Whether the model supports file attachments.
reasoning boolean Whether the model has native reasoning capabilities.
temperature boolean Whether the model supports temperature adjustments.
tool_call boolean Whether the model supports tool calling.
interleaved union Support for interleaved content. Can be true or an object specifying the reasoning field (reasoning_content or reasoning_details).
cost object Pricing per token for input, output, cache_read, and cache_write. Can include a context_over_200k override.
limit object Token limits for context, input, and output.
modalities object Supported input/output types (text, audio, image, video, pdf).
experimental boolean Marks the model as experimental.
status enum Current status: alpha, beta, or deprecated.
provider object Provider identifiers (npm package name or api endpoint).
options record Provider-specific configuration options.
headers record Custom HTTP headers to be sent with requests.
variants record Configuration for model variants, including a disabled flag.

Example: Complete Model Definition

Below is an example of a fully defined Model object as it would appear inside a provider's models record: 

{
  "id": "custom/super-model",
  "name": "Super Model 4K",
  "family": "Omni",
  "release_date": "2026-04-15",
  "attachment": true,
  "reasoning": true,
  "temperature": true,
  "tool_call": true,
  "interleaved": {
    "field": "reasoning_content"
  },
  "cost": {
    "input": 0.003,
    "output": 0.015,
    "cache_read": 0.0003,
    "cache_write": 0.0006,
    "context_over_200k": {
      "input": 0.006,
      "output": 0.030,
      "cache_read": 0.0006,
      "cache_write": 0.0012
    }
  },
  "limit": {
    "context": 200000,
    "input": 128000,
    "output": 4096
  },
  "modalities": {
    "input": ["text", "image", "pdf"],
    "output": ["text"]
  },

  "provider": {

2. Agent Configuration

Customize built-in agents or define new ones under agent.<name>.

Option Type Description
model string Override the global model for this specific agent.
prompt string Custom system instructions for the agent.
temperature number Randomness (0.0 = deterministic, 1.0 = creative).
top_p number Nucleus sampling parameter.
steps number Max agentic iterations before forcing a text response.
mode enum primary, subagent, or all.
hidden boolean If true, agent is hidden from the @ autocomplete menu.
color string UI color (Hex code or theme name like success).
permission object Agent-specific tool permissions.

Default Agent: Use default_agent to specify the starting primary agent (defaults to build).

Example: Agent Configuration

Below is an example of how to define a specialized agent in your opencode.jsonc:

{
  "agent": {
    "reviewer": {
      "model": "anthropic/claude-3-5-sonnet",
      "prompt": "You are a strict code reviewer. Focus on security, performance, and maintainability.",
      "temperature": 0.2,
      "top_p": 0.9,
      "steps": 5,
      "mode": "primary",
      "hidden": false,
      "color": "#ff5733",
      "permission": {
        "edit": "deny",
        "bash": "ask",
        "read": "allow"
      }
    }
  }
}

3. The Permissions System

Control what the AI can do with your system globally or per agent.

Permissions can be "allow", "deny", or "ask".

⚠️ Important: Order Matters
OpenCode uses a last-match-wins strategy. Rules are evaluated from the bottom up; therefore, rules defined later in your configuration object will override any matching rules defined earlier.

Available Keys

read, edit, glob, grep, list, bash, task, external_directory, webfetch, websearch, codesearch, lsp, skill, question, todowrite. 

"permission": {
  "edit": "allow",                // 1. Allow editing everything by default
  "edit": {
    "packages/core/*": "deny"     // 2. Override: deny editing core files (This wins for core files)
  }
}

4. Context Management (Compaction)

Manage session "memory" as it approaches the model's token limit.

Option Type Default Description
auto boolean true Automatically compact when context is full.
prune boolean true Erase old tool outputs to reclaim space.
reserved number 20000 Token buffer. Triggered at MaxContext - reserved.

Note on Calculation: The compaction trigger is based on the effective limit of the model. If you override a model's context limit in the provider configuration, the reserved buffer is subtracted from that new overridden limit to determine when auto-compaction begins.

5. Tools & Integrations

MCP (Model Context Protocol)

What it is: MCP is an open standard that enables AI models to seamlessly connect to external data sources and tools.

What it does: It acts as a standardized bridge between the LLM and "MCP Servers." These servers can expose databases, internal APIs, file systems, or third-party services (like Slack or GitHub) to the agent as tools.

When to use it: Use MCP when you need the agent to interact with data or functionality that is not provided by the built-in toolset. For example, if you want the agent to query a production database or fetch tickets from a project management tool.

Configure these under mcp.<name>. Local servers use a command; remote servers use a url.

LSP (Language Server Protocol)

What it is: LSP is a protocol that decouples the editor (OpenCode) from the language-specific intelligence (the Language Server).

What it does: It provides the agent with "IDE-like" capabilities, including precise go-to-definition, find-references, real-time type checking, and advanced refactoring, regardless of the language being used.

Default Included LSPs

OpenCode comes with a vast array of built-in LSPs. Some of the most common include:

Language LSP ID Extensions
TypeScript/JS typescript .ts, .tsx, .js, .jsx
Rust rust .rs
Go gopls .go
Python pyright .py, .pyi
C/C++ clangd .c, .cpp, .h, .hpp
Java jdtls .java
Bash bash .sh, .bash, .zsh
Nix nixd .nix
Svelte/Vue/Astro svelte, vue, astro .svelte, .vue, .astro

You can override or add custom servers under lsp.<id>.

Formatters

What it is: Formatters are tools that automatically rewrite source code to conform to a specific style guide (e.g., Prettier for JS, Black for Python).

What it does: It ensures that every change the agent makes to your codebase follows a consistent style, preventing "formatting noise" in your git diffs.

Set per-language formatting under formatter.<lang> by defining the command to run.

6. System & TUI Configuration

Option Type Description
logLevel enum debug, info, warn, error.
username string Custom name displayed in conversations.
snapshot boolean Track file changes for undo/redo (Default: true).
autoupdate union true, false, or "notify".

TUI Settings (tui.json)

Manage plugins and custom keybindings for the terminal interface.

7. Experimental Features

Toggle unstable features under the experimental key:

  • batch_tool: Enable grouped tool calls.
  • openTelemetry: Enable AI SDK telemetry spans.
  • primary_tools: Limit specific tools to primary agents.
  • continue_loop_on_deny: Continue agent loop even after a permission denial.
